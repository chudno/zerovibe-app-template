# Старт проекта: короткий онбординг → сразу вёрстка

Пользователь только что описал идею приложения (она пришла ПЕРВЫМ сообщением —
это его ответ на «что вы хотите создать?»). Твоя задача — быстро дособрать детали
и перейти к делу. Онбординг должен ощущаться как лёгкий разговор, а не анкета.

## Правила онбординга

- **НЕ переспрашивай идею.** Она уже есть в первом сообщении. Извлеки из неё
  максимум сам: тип приложения, вероятные экраны, возможное название. Подтверди
  своими словами, что понял, — и уточняй только то, чего не хватает.
- **Каждый вопрос — с дефолтом и опцией пропустить.** Пользователь — не технарь,
  ему нельзя «зависнуть» на вопросе. У любого пункта: предложи свой вариант +
  «или решим позже». Не блокируй прогресс ничьим молчанием.
- **Тупит — предлагай.** Не знает название / какие экраны — дай 2-3 конкретных
  варианта через `ask_user` (с пометкой рекомендуемого) или просто возьми
  разумный дефолт и скажи, что потом поменяем.
- Язык — продуктовый, без жаргона (skill `conventions`).

## Что выяснить (коротко, пакетом)

1. **Суть** — подтверди, что понял идею (1 фраза). Если правда неясно — один
   уточняющий вопрос.
2. **Название** — предложи 2-3 варианта под идею, или прими своё. «Решим позже» —
   поставь рабочее имя, продолжай.
3. **Настроение / цвета** — «на что должно быть похоже: строго и по-деловому,
   светло и дружелюбно, ярко и энергично?» → это выбор ТЕМЫ приложения (skill
   `aesthetics` — таблица настроение→тема DaisyUI). Один лёгкий вопрос, не больше.
   Не знает — возьми нейтральную тему.
4. **Главные экраны** — 2-4 ключевых экрана («что человек будет видеть: список
   заказов? форма заявки? личный кабинет?»). Предложи типичный набор под тип
   приложения. Это список для фазы 1.

Всё это — за 1-2 хода, не растягивай. Любой пункт можно отложить.

## Плавный переход → вёрстка

Как только ясен хотя бы ОДИН экран — не жди остального, **сразу переходи к фазе 1**
(skill `screens`): свёрстай первый экран с фикстурой, покажи в живом превью и спроси
«нравится?». Онбординг заканчивается ПЕРВЫМ показанным экраном, а не полной анкетой.
Остальные экраны выясняй и верстай по ходу.

## Границы

- Название/тему приложения ставишь ТЫ по итогам онбординга (это «дефолтная тема
  проекта»). Это не то же, что переключатель тем в шапке — тот уже есть для
  end-user'а, его не трогаем.
- В середине существующего проекта (экраны/код уже есть) онбординг НЕ нужен —
  сразу к задаче пользователя.
