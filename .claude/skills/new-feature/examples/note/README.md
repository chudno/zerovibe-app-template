# Эталон Note — рабочий пример среза через все слои

Это **справочный образец** одной сущности (`Note` — заметка пользователя),
проведённой через ВСЕ слои чистой архитектуры zerovibe. В самом шаблоне
приложения этого кода уже нет — он живёт здесь как эталон для skill `new-feature`.

Когда добавляешь новую сущность `X` (задачу, клиента, заказ), **сверяйся с этими
файлами**: они показывают точную форму каждого слоя, именование, обработку ошибок,
привязку к владельцу и регистрацию в админке. Копируй структуру, меняй имена под `X`.

## Файлы (по слоям)

| Файл | Слой | Что показывает |
|------|------|----------------|
| `domain.go` | domain | Структура сущности без тегов, конструктор-валидатор `NewNote`, инварианты через `ErrValidation` |
| `usecase.go` | usecase | Порт `NoteRepository` (интерфейс) + сервис `NoteService`: валидация → делегирование порту, без SQL/HTTP |
| `repository.go` | repository/sqlite | Реализация порта: записи через `db.Write`, чтения через `db.Read`, `INSERT ... RETURNING` |
| `repository_admin.go` | repository/sqlite | Admin-методы репозитория (список/поиск/фильтры/сортировка/CRUD для встроенной админки) |
| `adminres.go` | adminres | Дескриптор сущности для админки: поля, типы, `RegisterNote(...)` |
| `migration.sql` | platform/db/migrations | Схема таблицы отдельной goose-миграцией (`-- +goose Up/Down`) |
| `list.html.txt` | transport/web/templates | Шаблон страницы-списка (полная страница + HTMX) |
| `row.html.txt` | transport/web/templates | Шаблон фрагмента-строки (возвращается мутациями без перезагрузки) |

`.html.txt` — чтобы файлы не подхватывались `//go:embed` шаблонов приложения; при
переносе в проект переименовывай в `.html`.

## Ключевые приёмы, которые демонстрирует пример

- **Привязка к владельцу**: `OwnerID` + фильтр по `currentUser(r).ID` — так у каждого
  пользователя свои данные.
- **Очередь записи**: все мутации через `db.Write`, никогда в обход.
- **Миграция отдельным файлом**, а не inline `CREATE TABLE` в коде.
- **SPA без перезагрузки**: мутации возвращают HTML-фрагмент (`row`), а не редирект.
- **Регистрация в админке одной строкой** через дескриптор.

Подробный пошаговый порядок — в `../../SKILL.md` (skill `new-feature`),
нюансы полей админки — в skill `admin`.
