# Живое превью приложения

В облачном режиме человек видит твоё приложение в панели рядом с чатом.

**Превью полностью автоматическое — ты им НЕ управляешь и запускать его не нужно.**
Отдельная копия приложения работает рядом и показывает **последнюю СОХРАНЁННУЮ
версию** твоего кода.

> 🚫 **ЯЗЫК С ПОЛЬЗОВАТЕЛЕМ.** Без жаргона. НЕ говори «под», «контейнер»,
> «туннель», «сервер», «git», «коммит», «пересборка». Человеку: «глянь в окне
> справа», «сейчас обновлю». (См. skill `conventions`.)

## Когда превью обновляется — ГЛАВНОЕ

Превью показывает не то, что ты пишешь В ЭТУ СЕКУНДУ, а **последнюю версию,
которую ты СОХРАНИЛ** (`git push`, см. skill `conventions`). Пока ты не сохранил
правки — человек видит предыдущую версию.

Поэтому порядок такой:
1. Внёс правки, довёл до показуемого состояния, проверил что собирается (`make check`).
2. **Сохранил** (`git add -A && git commit -m "…" && git push`).
3. Через пару секунд превью подтянет новую версию — теперь приглашай смотреть.

**НЕ говори «глянь прямо сейчас» до того как сохранил** — человек увидит старое и
решит, что ты ничего не сделал. Сначала push, потом «готово, глянь в окне справа».

## Что от тебя нужно

- **Держи код собираемым** перед сохранением. Если код не компилируется — превью
  не сможет собрать новую версию и застынет на прошлой. Прогоняй `make check`
  (или `go build ./...`) и чини ошибки ПЕРЕД `git push`.
- **Сохраняй осмысленными кусками** и пуши, когда есть что показать (см.
  `conventions`). Каждый push = обновление превью.

Когда сделал заметное/визуальное — сохрани и пригласи взглянуть словами: «Готово,
глянь в окне справа». Панель обновится сама, ссылку давать не нужно.

## Самопроверка перед «готово»

Превью показывает СОХРАНЁННУЮ версию, а тебе надо проверить ТЕКУЩУЮ (ещё до show).
Не полагайся на панель превью для собственной проверки — подними приложение сам на
своём порту и проверь (см. skill `run`), потом сохраняй.

## Что знать (факты)

- Превью — **песочница для просмотра**, НЕ публикация. Данные в нём временные,
  сбрасываются. Постоянный адрес даёт отдельное действие «Опубликовать» (skill
  `deploy`).
- Превью показывает последнюю СОХРАНЁННУЮ версию. Забыл сохранить → человек видит
  старое. Сломал сборку → превью застыло на прошлой рабочей версии.
- Если превью в этом окружении недоступно (локальный прогон без облака) — просто
  работай как обычно, ничего специально не предлагай.
