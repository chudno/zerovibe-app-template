# Живое превью приложения (работает само)

В облачном режиме человек видит ЖИВОЕ приложение в панели рядом с чатом — то, что
ты пишешь прямо сейчас, ещё ДО публикации.

**Превью полностью автоматическое — ты им НЕ управляешь и запускать его не
нужно.** Оно поднимается вместе с тобой и обновляется само, как только ты правишь
код. Твоя единственная забота — держать код собираемым (об этом ниже).

> 🚫 **ЯЗЫК С ПОЛЬЗОВАТЕЛЕМ.** Без жаргона. НЕ говори «под», «контейнер»,
> «туннель», «сервер», «пересборка». Человеку: «глянь в окне справа», «обновится
> через пару секунд». (См. skill `conventions`.)

## Как это работает (чтобы ты понимал, а не управлял)

Рядом с тобой запущено твоё приложение — оно читает те же файлы, что ты правишь
(общая рабочая папка). За ним следит инструмент **air**: как только ты меняешь
`.go`-файл, air сам пересобирает приложение и перезапускает его (пара секунд).
Вёрстку (HTML-шаблоны, CSS) приложение подхватывает **мгновенно** с диска — без
пересборки. То есть:

- **Правка вёрстки/стилей** → человек видит сразу (F5).
- **Правка Go-логики** → человек видит через пару секунд (air пересобрал).

Тебе НЕ нужно ничего запускать, дёргать, обновлять или останавливать. Никаких
команд и эндпоинтов для превью нет.

## Что от тебя нужно — ровно одно

**Держи код собираемым.** Превью запускает твой ТЕКУЩИЙ код. Если он не
компилируется — air не сможет пересобрать, и человек будет видеть прошлую рабочую
версию (не новую). Поэтому перед завершением хода прогоняй `make check` (или хотя
бы `go build ./...`) и чини ошибки. Сломанная компиляция = превью «застыло» на
старом.

Когда сделал заметное/визуальное — просто пригласи взглянуть словами: «Обновил —
глянь в окне справа». Панель обновится сама, ссылку давать не нужно.

## ⚠️ Чего НЕ делать (иначе сломаешь превью)

- **НЕ запускай приложение сам** (`go run ./cmd/server`, `make dev/run`, свой
  процесс на порту 8080). Оно уже запущено рядом air'ом. Твой запуск займёт порт
  8080 → конфликт → живое приложение упадёт, и превью погаснет.
- **НЕ убивай процессы приложения**, не трогай порт 8080, не запускай долгие
  блокирующие процессы «сервером». Если тебе нужно ЧТО-ТО проверить самому —
  собери и погоняй на ДРУГОМ порту (`ADDR=:9090 go run ./cmd/server`) и обязательно
  останови после проверки. Порт 8080 — за air, не трогай.
- **НЕ трогай `/tmp`** приложения без нужды — там его песочная база превью.

Если всё же случайно уронил живое приложение (занял порт / убил процесс) — просто
освободи порт 8080 / заверши свой процесс; air сам поднимет приложение заново на
следующей правке или почти сразу.

## Что знать (факты)

- Превью — **песочница для просмотра**, НЕ публикация. Данные в нём временные.
  Постоянный адрес даёт отдельное действие «Опубликовать» (skill `deploy`).
- Превью показывает ТВОЙ текущий код из рабочей папки. Сломал сборку — человек
  видит старую версию; почини и скажи «поправил».
- Если превью в этом окружении недоступно (локальный прогон без облака) — просто
  работай как обычно, ничего специально не предлагай.
